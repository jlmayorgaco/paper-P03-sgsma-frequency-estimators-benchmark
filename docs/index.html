Project Scope

OpenFreqBench evaluates frequency estimator behavior under realistic disturbances with a canonical artifact contract. It supports full benchmark runs, targeted estimator/scenario slices, statistical tests, and deterministic report generation.

  • Pipeline core in src/pipelines/
  • Analysis core in src/analysis/
  • Benchmark plots in src/plotting/benchmark/
  • Paper integration in paper/

Quickstart

pip install -r requirements.txt
pip install -e .
openfreqbench env doctor
openfreqbench benchmark run
openfreqbench benchmark validate

Targeted execution modes

# one scenario vs all estimators
openfreqbench benchmark run-scenario --name IEEE_Freq_Step

# one estimator vs all scenarios
openfreqbench benchmark run-estimator --name EKF

# subset x subset matrix
openfreqbench benchmark run-matrix --scenario IEEE_Freq_Step,IEEE_Modulation --estimator EKF,UKF

# preview selection without execution
openfreqbench benchmark run-matrix --scenario IEEE_Freq_Step --estimator EKF --dry-run

Release Quality Gates

  • openfreqbench quality-gate --profile canonical
  • openfreqbench quality-gate --profile legacy
  • openfreqbench quality-gate --profile manual-nightly

Canonical output validation: openfreqbench benchmark validate

Why this benchmark

Grid dynamics in low-inertia systems require frequency estimators that balance latency, robustness, and numerical stability. OpenFreqBench was built to make those trade-offs auditable, repeatable, and publication-ready.
